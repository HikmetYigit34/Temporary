A Real-Life Example
    Data collection, processing data and taking corrective action
Goal
    Identify certain mistakes on multiple design drawings, created by multiple designers, and inform each designer which mistakes on which drawing he/she did to make corrective action. These drawings are in development stage so they are changing like rev1, 2, 3, .... n.
    Challenge myself if I can create an automated solution for capturing "certain mistakes".
Inputs
    I have around 20 group of drawing each have around 5 pages, so total of around 100 drawings. A small cut out from one DRAWING shown below as an example.
    For equipment tagging on DRAWINGS certain CODES should be used, we (the company) have a Master List for those codes; In other words, designer can only use those CODES from Master List, cannot create on his/her own. Here below is how Master List is look like. It has around 1000 items (codes). Text parts collected from drawings are about 200,000 items, after cleaning, codes are about 20,000 So, we are comparing 20,000 items to 1000 items roughly, to give you an idea about magnitude of the job.
    Main idea
    Create a table, CODE versus DRAWINGS,
    and highlight mistakenly used codes on drawings.
    Then inform that drawing creator where he/she did mistake(s) to take action to correct the mistake.
Steps to Solution
    Using traditional tools,
    Adobe (drawings are in pdf format), export text only
    Excel and its standard functions, VBA, Pivot Table,
  1. Collect data from drawings
  2. Take only readable text from data, remove all spaces and line breaks
  3. Convert text to a string, each page is only one single string
  4. Split string by "-", dash
  5. List split text parts
  6. Collect only -####- 4 character in between dashes
  7. Collect only -####- 4 alpha, ignore numerics
  8. Pivot Table, rows are Codes, cols are Drawings/Pages, Drawing here is filename/pageNum of the drawing
Final Output (With traditional tools and approach)
    Below table shows BCV is wrong (not in approved codes list) but used in G20 drawing, and so on...
    We can now, inform G20 designer and let him/her know what are the mistakes on his/her drawing to take a corrective action...
    This table size is 60 cols and 600 rows.
